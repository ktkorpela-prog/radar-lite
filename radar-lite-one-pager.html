@essentianlabs/radar-lite is a local, open-source npm package that scores what your agent intends to do — before it does it. It returns one of three verdicts: PROCEED, HOLD, or DENY. Runs entirely on your machine. Zero telemetry, no data sent to EssentianLabs. Ships Vela Lite, a lightweight local version of our risk-management Essentian.

The three verdicts
PROCEED
T1 · score 2
Read the customer's order status from the internal CRM.
HOLD
T2 · score 16
Send newsletter to 50,000 users. Staged rollout before full send.
DENY
Deterministic
Delete all payment records. Score 22 + irreversibility signal.

RADAR returns a verdict — your code decides what to do with it. It does not block actions by itself. A PROCEED is not approval, authorisation, or certification. Liability stays with you.

How it works, tier by tier
T1
LOW RISK
Rules engine + short LLM verdict. One line, e.g. "Proceed, internal read-only action."
T2
ELEVATED
LLM assesses and returns four strategy options — avoid, mitigate, transfer, accept — with a recommended choice.
DENY
HARD STOP
Triggered deterministically — either by a policy deny pattern, or score 20+ combined with an irreversibility signal.
Get started — five minutes
  1. Install Node.js 18+ if you don't have it.
  2. Install the package:
    npm install @essentianlabs/radar-lite
  3. Get an LLM API key — Anthropic, OpenAI, or Google.At least one required for Vela Lite. Rules engine works without.
  4. Configure in ~/.radar/.env or via radar.configure().
  5. Open the local dashboard:
    npx radar-lite dashboard → http://localhost:4040/lite
  6. Seed demo data:
    npx radar-lite demo
  7. Wire into your agent using the integration cookbook in the README.
▸ MINIMAL EXAMPLE
// one function call before your agent acts
import { radar } from '@essentianlabs/radar-lite'

const result = await radar.assess({
  action: "Send invoice for £12,400",
  activityType: 'financial'
})

if (result.verdict === 'PROCEED') {
  executeAction()
} else {
  holdForReview(result.tier)
}

Privacy & security
  • Light security audit complete — 0 known vulnerabilities (npm audit)
  • Dashboard bound to 127.0.0.1 only — not network-accessible
  • Action text never stored — only SHA256 hashes in local SQLite
  • LLM calls go direct from your machine to your chosen provider, using your key
  • No telemetry. No phone-home. No EssentianLabs servers in T1/T2
  • HTML escaping on dashboard data (XSS prevention)
  • LLM error messages sanitised — no provider leakage
  • Rate limiter on HTTP API — 100 req/min
  • Beta software — not for production without independent legal/compliance review
